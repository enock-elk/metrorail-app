🏗️ System Architecture

Core Stack

  • Frontend: Vanilla JS + Tailwind CSS (via CDN).
  • Data Pipeline (Waterfall): Cloudflare Edge (Primary) ➔ GitHub CDN (Fallback) ➔ Firebase RTDB (Failsafe).
  • Local Cache: IndexedDB (NextTrainDB) with RAM fallback.
  • Rendering Engine: SPA currently, transitioning to Astro "Islands" Architecture (Zero-JS SSG for schedules).
  • Routing: Cloudflare DNS (Proxied) + Cloudflare Edge Workers (Telemetry & Social Bots).

File Structure

  • index.html: App shell & Analytics container.
  • js/logic.js: Data fetching, state & Holiday logic.
  • js/planner-core.js: Trip planning (Dijkstra + Hub Bans).
  • js/config.js: Routes, Regions (GP/WC/KZN/EC), Fares & Constants.
  • map.html: Interactive Map (Hybrid Engine: Dynamic + Static).
  • status.html: Live system health and roadmap tracker.
  • marketing.html: Printable A5 flyers and social assets.
  • service-worker.js: Offline caching logic (Cache-First/Stale-While-Revalidate).

🚨 Critical Protocols (Non-Negotiable)

1. Analytics Inviolability

The Google Analytics script (ID: G-JM5DH6ERVX) in index.html is the proof of traction. NEVER remove or comment it out.

2. The "Index 0" Trap

Firebase JSON often has metadata at Index 0. NEVER use .shift(). Use filter() to remove rows starting with "Last Updated".

3. Hybrid Planner Logic

The Planner uses a "Hybrid" model (Smart Pruning + Path Validation) to handle complex hubs like Pretoria. DO NOT re-introduce strict directionality checks.

4. Storage Safety (Crash Immunity)

iOS Private Browsing strict limits cause QuotaExceededError. NEVER use raw localStorage.setItem(). Always use the safeSetLocalStorage() wrapper in ui.js.

5. Hybrid Map Engine

map.html parses dynamic coordinates but falls back to STATIC_ROUTE_PATHS. NEVER remove the static fallbacks. They guarantee 100% offline rendering if the DB lacks coordinates.

6. Telemetry Integrity

Grid analytics use a 2-second debounce to prevent CSS animation double-fires. NEVER bypass the _lastGridTrackTime lock. App Version tags must always append to events.

7. Firebase Export Sanitization

Raw Firebase JSON exports MUST be scrubbed of admin_state, feedback, sys_logs, and inbox nodes. NEVER commit private nodes to public GitHub to prevent massive PII data leaks and RAM bloat.

8. Ad-Blocker Evasion

Telemetry beacons to Firebase MUST use PUT requests directed to /sys_logs/ instead of POST to /telemetry/. NEVER use "telemetry" or "analytics" in paths to evade regex filters and restrictive rules.

9. WebView Breakouts

Use Android intent:// links and iOS "Three Dots" UI guides. ALWAYS rip users out of Facebook/IG captive browsers to secure the PWA installation natively in Safari/Chrome.

10. Lie-Fi Deadlock Guard

guardianFetch must use AbortControllers. ALWAYS cleanly differentiate deliberate route swaps from genuine network timeouts to prevent permanent offline-state deadlocks.
